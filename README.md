# 🛒 Управление корзинами пользователей (Frontend Test Task)

SPA-приложение для просмотра и редактирования корзин пользователей, работающее на публичном API
[DummyJSON](https://dummyjson.com/docs/carts).

В проекте реализованы просмотр списка корзин с пагинацией, фильтрация по пользователям, детальный просмотр корзины и возможность её
редактирования (изменение количества товаров, удаление) с синхронным обновлением локального кэша.

## 🛠 Технологический стек

- **Ядро:** React 18, TypeScript, Vite
- **Управление серверным состоянием:** `@tanstack/react-query` v5 + Axios
- **Глобальное состояние (UI):** Zustand
- **Роутинг:** React Router v6
- **Стилизация:** `@emotion/styled`
- **Архитектурная методология:** Feature-Sliced Design (FSD)

## 🏗 Архитектура (Feature-Sliced Design)

Проект строго следует архитектурной методологии **FSD**, что обеспечивает высокую масштабируемость, предсказуемость и независимость модулей.

Используемые слои (от высшего к низшему):

- `app/` — Инициализация приложения, глобальные провайдеры (QueryProvider, RouterProvider) и глобальные стили.
- `pages/` — Композиционный слой. Страницы собирают виджеты и фичи вместе (CartsPage, CartDetailsPage).
- `widgets/` — Самостоятельные блоки интерфейса, связывающие данные и логику (CartListWidget, CartDetailedWidget).
- `features/` — Пользовательские сценарии и бизнес-логика (CartPagination, CartFilters, CartItemActions).
- `entities/` — Бизнес-сущности. Здесь хранятся доменные модели (Types), DTO, мапперы и API-запросы для `Cart`.
- `shared/` — Переиспользуемый код: UI-кит (Button, Loader), инстанс API (Axios), глобальные хранилища Zustand (store).

**Ключевые архитектурные решения:**

1. **Разделение DTO и Domain:** Данные от DummyJSON проходят через слой мапперов в `entities/cart/lib`, благодаря чему UI-компоненты
   работают с чистыми доменными моделями, не зависящими от контрактов бэкенда.
2. **Оптимистичное обновление кэша:** Так как DummyJSON — это mock-сервер и он не сохраняет PUT-запросы в реальной базе, реализовано ручное
   обновление кэша через `queryClient.setQueryData` и `setQueriesData`. При редактировании корзины данные мгновенно обновляются как на
   детальной странице, так и в таблице на главной странице.
3. **Бесшовный UX:** Для пагинации используется `placeholderData: keepPreviousData`, что предотвращает "прыжки" интерфейса при переключении
   страниц.

## 🚀 Инструкция по запуску

Для запуска проекта локально потребуется Node.js и пакетный менеджер `yarn` (или `npm`).

**1. Клонирование репозитория:** \`\`\`bash git clone <ссылка-на-твой-репозиторий> cd carts-project \`\`\`

**2. Установка зависимостей:** \`\`\`bash yarn install \`\`\`

**3. Запуск dev-сервера:** \`\`\`bash yarn dev \`\`\`

Приложение будет доступно по адресу: [http://localhost:5173](http://localhost:5173)

**4. Сборка для продакшена:** \`\`\`bash yarn build \`\`\`

## 🧩 Реализованный функционал

- ✅ **Список корзин:** Таблица с основной информацией, серверная пагинация, выбор лимита (5/10/20).
- ✅ **Фильтрация:** Поиск корзин по `userId` (реализован паттерн Debounce для инпута).
- ✅ **Детальная страница:** Просмотр товаров в конкретной корзине.
- ✅ **Редактирование:** Изменение количества товара (+ / -) и удаление товара.
- ✅ **Сохранение стейта:** При переходе между страницами (список ↔ детали) состояние пагинации и фильтров сохраняется благодаря Zustand.
- ✅ **Обработка состояний:** Проработаны состояния Loading, Error, Disabled, Hover для всех интерактивных элементов.

## debt Технический долг и точки роста (Что можно улучшить)

В рамках ограниченного времени на выполнение тестового задания были допущены некоторые архитектурные упрощения. Для вывода проекта на

- **Устранение хардкода и "магических чисел":** Вынести базовый URL API (`https://dummyjson.com`), задержку Debounce (`500ms`), опции
  лимитов пагинации (`[5, 10, 20]`) и дефолтные значения в слой конфигурации (`shared/config/constants.ts`) и переменные окружения (`.env`).
- **Строгая типизация:** Заменить строковые литералы на `enum` или `const`-словари (например, для путей роутера, ключей React Query и
  HTTP-методов), чтобы исключить опечатки и улучшить автокомплит.
- **Покрытие тестами:** Добавить Unit-тесты (с использованием **Vitest** + **React Testing Library**) в первую очередь для чистых функций
  (мапперов в `entities/cart/lib`) и хуков, а затем покрыть интеграционными тестами ключевые пользовательские сценарии.
- **Глобальная обработка ошибок:** Внедрить `ErrorBoundary` для перехвата крашей React-дерева, а локальные текстовые сообщения об ошибках
  HTTP-запросов заменить на глобальную систему уведомлений (Toasts/Snackbars).
- **Инфраструктура и Code Quality:** Настроить строгие правила ESLint (с плагинами для FSD, если потребуется) и Prettier, а также прикрутить
  Git Hooks (**Husky** + **lint-staged**) для запрета коммитов с неформатированным кодом или TypeScript-ошибками.
- **Доступность (a11y):** Провести аудит и добавить необходимые `aria-` атрибуты, улучшить фокусы для полноценной навигации с клавиатуры и
  поддержки скринридеров.
